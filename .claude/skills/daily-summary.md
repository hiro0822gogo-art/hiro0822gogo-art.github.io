# daily-summary スキル

## 手順

1. `git log --oneline --since="midnight"` で今日のコミット一覧を取得する。
2. `git diff HEAD~N HEAD` または各コミットの diff を確認し、変更内容を把握する。
3. 以下のフォーマットで日本語まとめを出力する。

## 出力フォーマット

```
# 今日の作業まとめ（YYYY年M月D日）

## やったこと
- （変更内容を箇条書きで。技術用語は簡単な説明を添える）

## 変更ファイル
- `ファイル名` — 何をしたか一言

## 次にやること（あれば）
- （残タスクや気になる点）
```

## 注意

- コミットがない場合は「今日のコミットはありません」と伝える。
- 変更が多い場合は、ファイル単位ではなく目的・機能単位でグループ化して要約する。
- 専門用語は使わず、非エンジニアにも伝わる言葉でまとめる。
